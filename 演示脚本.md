# 随景 · SuiJing — 2分钟演示脚本

## 开场（0:00 - 0:15）

> 大家好，我做的产品叫「随景」—— 一个基于 GPS 路线的沉浸式声景生成器。
>
> 核心理念很简单：当你开车驶过不同的地形，音乐和环境音会自动跟着变化。

## 展示路线（0:15 - 0:30）

> 现在大家看到的是一条深圳的模拟路线，从海滨公路出发，经过山区、城市、高速，最后穿越隧道。
>
> 五段路程，五种场景，每段都有独立的背景音乐和环境音效。

**操作：指向地图上的五段彩色路线**

## 点击出发（0:30 - 1:00）

> 点击「开始行驶」。

**操作：点击按钮**

> 可以看到小车开始沿路线移动，海浪声和 Lo-fi 音乐同时响起。
>
> 注意顶部的进度条和场景导航，当前高亮的是「海滨公路」。

**等待小车移动几秒**

## 场景切换（1:00 - 1:30）

> 现在小车进入了「山区公路」—— 注意听，音乐从 Lo-fi 无缝淡入了 Ambient Folk，环境音也从海浪变成了风声。
>
> 这个切换用的是 Web Audio API 的交叉淡入淡出，1.5秒完成，完全无感。

**指向场景卡片**

> 右边的场景卡片实时更新了场景名和音乐风格。

## AI 亮点（1:30 - 1:50）

> 这是最关键的亮点：每次场景切换时，AI 会实时生成一段氛围描述。

**指向 ✨ AI 区域**

> 看，文字正在逐字打出来 —— 这是真正的 AI 实时生成，不是预设文案。
>
> 而且它还接入了和风天气 API，今天深圳是晴天，所以标签显示「晴·明快」，AI 的描述也会融入天气因素。如果是雨天，整体氛围会变得更低沉。

## 总结价值（1:50 - 2:00）

> 总结一下技术栈：Next.js + 高德地图 + Web Audio API + OpenAI + 和风天气。
>
> 「随景」证明了一件事：AI 可以让驾驶体验从「听歌」升级为「沉浸式声景」，每一段路途都有独一无二的音乐氛围。
>
> 谢谢大家。

---

## 演示要点提醒

- 演示前确保音量适中，能听到音乐和环境音的切换
- 手机竖屏展示效果更佳（已做移动端适配）
- 如果 AI 文案没出来，卡片仍会显示降级文案，不影响演示流程
- 可以中途暂停，指着卡片讲解细节
